Muy bien. Es hora de sumergirme en mi servicio administrado favorito de AWS: Lambda. Lambda es un servicio de cómputo sin servidor. También se conoce como cómputo Serverless.
Supongamos que estás construyendo una aplicación clasificadora de cangrejos donde los usuarios suben una imagen de un crustáceo y luego reciben una notificación cuando se ha realizado la clasificación. Después de que tu aplicación ha sido programada, necesitas instalarla en una infraestructura. Esto significa aprovisionar servidores, escalar hacia arriba y hacia abajo y asegurarte de que todo esté siempre disponible.
Puedes usar Lambda para ejecutar código. No necesitas pensar en servidores o clústeres de cangrejos. Creas una función Lambda, pones tu código ahí, configuras un disparador y tu función se ejecuta en respuesta a ese disparador.
Volviendo a la aplicación clasificadora de cangrejos. Un disparador simple podría ser cuando un usuario sube una nueva imagen de un cangrejo o cuando la aplicación ha determinado la clasificación de una imagen subida. Sin embargo, los disparadores pueden ser más complejos como procesar datos en tiempo real desde un flujo o redimensionar una imagen en diferentes resoluciones.
Y la mejor parte es que no tienes que administrar el entorno. AWS lo maneja todo por ti. Este entorno administrado es automáticamente escalable y altamente disponible. Esto significa que ya sea que recibas un solo disparador o 1000, Lambda escalará para satisfacer la demanda. Mejor aún, AWS se encarga de todas las actualizaciones de parches y seguridad. Simplemente te enfocas en tu código y con ese código sólo debes tener en cuenta que la duración máxima de una función Lambda es 15 minutos.
Mira, no estamos siendo egoístas, pero si puedes dividir tu código en segmentos menores a 15 minutos, Lambda podría ser la mejor opción. Lambda es excelente para procesos rápidos basados en eventos, por ejemplo, manejar solicitudes de sitios web, procesar lotes de datos y generar informes de gastos.
¿Y mencioné que Lambda admite cualquier lenguaje de programación? ¿Por qué lo hace? Lambda admite varios lenguajes de programación de forma predeterminada mediante el uso de entornos de ejecución, lenguajes como Java, Python, Node.js. Sin embargo, también puedes crear tu propio entorno de ejecución personalizado. Un entorno de ejecución proporciona un entorno específico de lenguaje que transmite eventos de invocación, información de contexto y respuestas entre Lambda y la función.
Una última cosa que mencionar es que Lambda se integra con otros servicios de AWS con bastante facilidad. Esto significa que puedes construir tu aplicación clasificadora de cangrejos completamente desarrollada, utilizando una gran cantidad de servicios de AWS y aun así, no tener que preocuparte por iniciar o detener servidores. Sólo tienes que preocuparte por poner todas esas especies de cangrejos en tu base de datos. Por cierto, hay más de 7000 especies de cangrejos.
