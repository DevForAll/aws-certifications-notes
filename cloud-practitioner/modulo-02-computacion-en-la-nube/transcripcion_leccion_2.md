Ahora que hemos aprendido sobre EC2, vamos a profundizar y a hablar sobre los tipos de instancias. Visualiza las instancias de EC2 como diferentes tipos de máquina de café en nuestra cafetería. A los clientes les gusta la variedad. No todos beben lo mismo, así que necesitaremos más de un tipo de máquina de café que de otra.
Por ejemplo, necesitamos una máquina de espresso de alta potencia para aquellos que quieren shots de espresso o lattes. También necesitaremos una máquina de café de goteo clásica para los clientes que sólo quieren un café básico. O tal vez una máquina de vapor para aquellos que quieren un chocolate caliente. Por lo tanto, si queremos que nuestro negocio opere de la manera más eficiente posible, es importante usar la máquina de café adecuada para cada pedido.
Esto es similar a implementar instancias de EC2. Hay diferentes tipos de instancia que sirven para diferentes propósitos en tu entorno de AWS. Los tipos de instancia se agrupan en cinco familias de instancias que ofrecen diferentes combinaciones de CPU, memoria, almacenamiento, capacidad de red. Esto te da la flexibilidad para optimizar para ciertos tipos de tareas, eligiendo instancias específicas para tu carga de trabajo particular.
Las diferentes familias de instancia son: uso general, optimizadas para cómputo, optimizadas para memoria, computación acelerada y optimizadas para almacenamiento.
Las instancias de uso general proporcionan un buen equilibrio de recursos de cómputo, memoria y redes. Pueden usarse para muchas cargas de trabajo diversas, como servicios web o repositorios de código. También son un buen punto de partida si no sabes cómo se comportará tu carga de trabajo con anticipación.
Las instancias optimizadas para cómputo son ideales para tareas que requieren mucho procesamiento, como servidores de juegos, computación de alto rendimiento, tareas de aprendizaje automático e incluso modelado científico.
Las instancias optimizadas para memoria son buenas para tareas intensivas en memoria. Ofrecen un rendimiento rápido para cargas de trabajo que procesan grandes conjuntos de datos en memoria RAM.
Las instancias de computación acelerada son buenas para cálculos de punto flotante, procesamiento gráfico o coincidencia de patrones de datos. Esto se debe a que utilizan aceleradores de hardware. Estos son coprocesadores que realizan funciones de manera más eficiente que los que es posible en software ejecutándose en CPU.
Y finalmente las instancias optimizadas para almacenamiento son ideales para cargas de trabajo que requieren alto rendimiento para datos almacenados localmente.
Después de decidir sobre un tipo de instancia, la siguiente parte es elegir el tamaño adecuado. Te insto a tener en cuenta no sólo el rendimiento, sino también el costo. Claro, las instancias más grandes te dan más CPU, más memoria, más almacenamiento, pero también cuestan más. Por lo tanto, es importante encontrar un equilibrio que se ajuste a tus necesidades y presupuesto. Esto asegura que elijas el tamaño de instancia adecuado para obtener el mejor rendimiento, pero sin pagar por recursos que realmente no necesitas.
La mejor parte es que no tienes que usar un tipo o tamaño de instancia específico para siempre. Podrías descubrir que el que elegiste inicialmente no ofrece el rendimiento que necesitas. Esto está perfectamente bien. Puedes cambiarlo. La nube te da la capacidad de cambiar muy rápidamente, así que aprovéchala.
