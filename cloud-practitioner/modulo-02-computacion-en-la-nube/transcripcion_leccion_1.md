En este video vamos a hablar a alto nivel sobre un servicio llamado Amazon Elastic Compute Cloud o Amazon EC2. Si recuerdas nuestra cafetería, los empleados son una metáfora para el modelo cliente-servidor, donde un cliente envía una solicitud al servidor. El servidor trabaja y luego envía una respuesta. Este ejemplo es para la cafetería, pero la misma idea se aplica a otros negocios.
Estos negocios, ya sea que estén en el sector de la salud, manufactura, seguros o entregando contenido de video, también están utilizando este modelo para entregar productos, para entregar recursos o datos a los usuarios finales. Estos negocios necesitan capacidad de cómputo, capacidad de cómputo básica para alojar sus aplicaciones y proporcionar la potencia de cómputo que su negocio necesita.
Cuando trabajas con AWS, esos servidores se llaman instancias EC2. Las instancias de EC2 son máquinas virtuales en la nube, pero en AWS no lo vamos a decir máquinas virtuales, sino que le diremos instancias. Utilizar EC2 para el cómputo es altamente flexible, rentable y rápido cuando lo comparas con ejecutar tus propios servidores en las instalaciones de un centro de datos en tu negocio o empresa.
El tiempo y dinero que toma poner en marcha recursos en las instalaciones es bastante alto, pero con EC2 es mucho más fácil comenzar a trabajar. AWS se encargó de la parte difícil por ti y AWS está constantemente operando una cantidad masiva de capacidad de cómputo lista para ser utilizada, y puedes usar cualquier porción de esa capacidad cuando la necesites.
Todo lo que tienes que hacer es solicitar las instancias EC2 que deseas y se iniciarán y arrancarán y estarán listas para ser utilizadas en pocos minutos. Después de que hayas terminado, puedes detener o terminar las instancias EC2. No estás encerrado o atascado con servidores que no necesitas o que no quieres. Tu uso de instancias EC2 puede variar enormemente con el tiempo y sólo pagas por lo que usas porque con EC2 sólo pagarás por las instancias en ejecución, no por las instancias detenidas o terminadas.
Como mencioné, las instancias EC2 son máquinas virtuales. Las máquinas virtuales comparten un host físico subyacente con otras instancias, lo cual es un concepto llamado multitenencia o multi tenancy en inglés. Esto quiere decir que en una misma computadora física vas a tener muchas máquinas virtuales usando software especializado para esto.
En un entorno multihost es necesario asegurar que cada máquina virtual esté aislada de las demás, pero que aún pueda compartir los recursos proporcionados por el host físico. Este trabajo de compartir recursos y aislamiento lo está haciendo una pieza de software llamada hipervisor, que se ejecuta en la máquina host. Para el servicio de EC2, AWS administra ese host subyacente, administra al hipervisor y el aislamiento de instancia a instancia. Así que aunque no estarás administrando esta parte, no tendrás que preocuparte por esto. Pero es importante tener una comprensión básica del concepto de multitenencia.
Cuando aprovisionas una instancia EC2, puedes elegir el sistema operativo basado en Windows o Linux. Puedes también aprovisionar miles de instancias EC2 bajo demanda, con una mezcla de sistemas operativos y configuraciones para alimentar las diferentes aplicaciones de tu negocio. También puedes configurar qué software quieres ejecutar dentro de esa instancia, ya sean tus propias aplicaciones internas de negocio o aplicaciones web simples o complejas, bases de datos, software de terceros, paquetes de software empresarial. Tú tienes control completo sobre lo que sucede en esa instancia.
En una próxima demostración veremos cómo lanzar una instancia EC2. Las instancias EC2 también son redimensionables. ¿Recuerdas la palabra "elastic" en el nombre de EC2? Pues significa que puede cambiar de tamaño. Puedes comenzar con una instancia pequeña y darte cuenta de que la aplicación que estás ejecutando está comenzando a necesitar más recursos de ese servidor. Entonces puedes modificar la instancia para que tengas más memoria y más CPU. Esto es lo que llamamos escalar verticalmente o escalamiento vertical para una instancia. En esencia, puedes hacer que las instancias sean más grandes o más pequeñas cuando lo necesites.
También controlas el aspecto de redes de EC2. Vas a tener control sobre qué tipos de solicitudes llegan a tu servidor y si son accesibles de manera pública o privada. Hablaremos más sobre esto en la sección de redes.
Ahora es importante notar que el concepto de máquina virtual no es algo nuevo. AWS simplemente lo ha hecho mucho más conveniente y rentable para que adquieras servidores a través de este modelo de cómputo como servicio.
