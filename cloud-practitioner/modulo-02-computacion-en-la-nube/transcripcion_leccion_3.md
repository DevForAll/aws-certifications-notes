Hemos estado hablando sobre diferentes servicios de AWS, así como de la infraestructura global. Quizás te estás preguntando cómo interactúo realmente con estos servicios. La respuesta es a través de APIs. Las siglas API vienen de Application Program Interface, en español es una interfaz de programación de aplicaciones que define formas predeterminadas para interactuar con los servicios programáticos. En todo es una llamada a una API. Puedes invocar o llamar a estas APIs para aprovisionar, configurar y administrar tus recursos.
Hay tres formas principales de llamar a la API de AWS: usando la consola de administración de AWS, la interfaz de línea de comandos de AWS o CLI o el kit de desarrollo de software (SDK) de AWS.
Primero hablemos sobre la consola que está basada en un navegador. Usando la consola, puedes administrar tus recursos visualmente y de una manera fácil de entender. Esto es excelente para comenzar y desarrollar tu conocimiento de los distintos servicios. También es útil para configurar entornos de prueba, ver facturas de AWS, monitorear recursos y administrar tareas no técnicas. La consola es probablemente el primer lugar al que irás cuando estés aprendiendo sobre AWS.
Sin embargo, después de estar operando en un entorno de tipo producción, no querrás continuar haciendo todos estos clics dentro de la consola. Por ejemplo, para crear una instancia EC2 necesitas navegar a través de varias pantallas, establecer todas las configuraciones que deseas y luego lanzar tu instancia. Si quieres lanzar otra instancia EC2 más tarde, tendrías que volver a la consola y navegar nuevamente por esas pantallas para ponerla en funcionamiento. Al hacer que los humanos realicen este tipo de aprovisionamiento manual, te expones a posibles errores. Es muy posible olvidar seleccionar una casilla o escribir algo mal cuando estás haciendo todo manualmente.
Entonces puedes pensar en utilizar la CLI de AWS (Command Line Interface) para invocar la API de AWS usando una terminal. Esto es diferente al estilo de navegación visual de la consola web porque te permite interactuar con los servicios de AWS a través de una entrada basada en texto basada en comandos. Esto hace posible la automatización mediante scripts.
Veamos dos ejemplos de invocación de APIs de AWS usando la interfaz de línea de comandos. Podemos ejecutar comandos usando AWS Cloud Shell, que es una terminal basada en la nube que ya tiene la CLI de AWS instalada en un entorno administrado. En este ejemplo vamos a crear una instancia EC2 programáticamente con el comando AWS EC2 instances. Ejecutamos el comando y la instancia ahora se está inicializando. Y si quieres hacer algo más como listar todas las zonas de disponibilidad en la región actual, podrías usar el comando AWS EC2 describe.
Estos son dos ejemplos básicos, pero a medida que explores más servicios y recursos de AWS, descubrirás comandos adicionales de la CLI de AWS para interactuar con esos servicios también. También quiero señalar que estos comandos en este ejemplo se ejecutaron manualmente, pero pudieran incluirse en scripts y otros procesos de automatización. La automatización es un aspecto importante para tener una implementación en la nube exitosa y predecible a lo largo del tiempo.
Muy bien. Otra forma de interactuar con AWS es a través del SDK de AWS. El SDK de AWS hace posible que interactúes con los recursos a través de varios lenguajes de programación, como por ejemplo Python. Ejecutamos un script de Python usando el entorno de desarrollo integrado de Visual Studio Code que utiliza el SDK para listar las instancias EC2 en la región actual.
Así que a medida que comienzas a trabajar con más servicios, recuerda que AWS aloja APIs que puedes utilizar para crear, modificar, eliminar, interactuar y administrar todos los recursos, ya sea que estés usando la consola de administración, la CLI o el SDK de AWS. Estas APIs se están llamando detrás del escenario.
