Has estado aprendiendo mucho sobre EC2 y probablemente ya estás pensando cómo puedo crear una instancia EC2 por mí mismo. En este video vamos a recorrer el proceso de creación de una instancia EC2 para un servidor web utilizando la consola de administración de AWS. Revisaremos algunas configuraciones clave en el camino, así que comencemos.
Lo primero que queremos hacer es ir a la consola EC2. Ahora resulta que la tengo aquí en mis visitados recientemente y en mi barra de accesos directos, pero fácilmente podría buscar EC2 e ir al servicio. Hay muchas opciones aquí, pero vamos a centrarnos específicamente en la opción de lanzar instancia.
Lo primero que debemos elegir es su nombre. Tenemos que elegir nuestro nombre de instancia para poder encontrarlo más tarde. Lo siguiente que vamos a hacer es elegir la imagen de Amazon Machine Image o AMI. Es una plantilla del sistema operativo y las aplicaciones integradas con las que vendrá. Podemos personalizarla, pero por ahora vamos a usar la predeterminada, la AMI de Amazon Linux. Esta AMI de Amazon Linux es perfecta para un servidor web de uso general.
Lo siguiente que debemos decidir es el tipo de instancia. Esto se refiere a cuánta potencia de cómputo tendrá nuestro servidor web. En este caso, vamos a elegir sólo el T2 micro básico. Revisaremos algunos de los detalles que componen este T2 micro un poco más tarde, pero por ahora esto estará bien. Una CPU virtual, un gigabyte de memoria y está en la capa gratuita, lo cual me parece bien.
Lo siguiente que elegimos es el par de claves y esto está relacionado con cómo vamos a iniciar sesión en esta instancia EC2. Un par de claves específicamente: una clave pública que se incluirá en la instancia EC2 y una clave privada que conservaremos. Podemos crear este par de claves aquí mismo, pero voy a elegir una clave que ya teníamos establecida.
La siguiente sección es elegir la configuración de red. Nos divertiremos mucho revisando todos estos detalles un poco más tarde, pero por ahora basta decir que vamos a permitir el tráfico HTTP de Internet. Después de todo, será un servidor web. Así que esto es todo lo que tenemos que hacer.
La siguiente opción que tenemos que elegir son las opciones de almacenamiento para nuestra instancia EC2. En este caso, le daremos ocho gigabytes de espacio en disco utilizando el volumen GP3 EBS o volumen de Elastic Block Store. Revisaremos lo que eso significa un poco más tarde, pero esto significa que tendrá suficiente espacio para servir páginas web.
Y hablando de servir páginas web, tenemos que hacer un cambio más para que realmente sea un servidor. Cuando elegimos la AMI, elegimos una AMI muy genérica. Esta no tiene el servidor web activado al inicio. Para hacer eso, vamos a ir a los detalles avanzados, específicamente a la sección de datos de usuario. Lo que nos permite hacer los datos de usuario es pegar un script como este que instalará y activará el servidor web Nginx, que es lo que vamos a utilizar para servir contenido de Internet.
Todo esto se ve bien, así que voy a presionar "lanzar instancia" y veremos qué obtenemos. Tan pronto como se lance la instancia, podemos ir a su consola web y ver algunos detalles sobre esa instancia EC2.
Ahora que nuestra instancia EC2 está funcionando, voy a copiar su dirección IP pública, voy a abrir un nuevo navegador y vamos a ella. Y ahí lo tienes: tu propia instancia EC2 ejecutando un servidor web básico. ¡Felicidades!
Eso es lo básico para configurar instancias EC2 y vamos a entrar en más detalles sobre los matices más adelante en el curso, así que mantente atento.
